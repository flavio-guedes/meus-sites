Manim
Python · Animação Matemática
⏱ Tempo & Ritmo
run_time
Controla quanto tempo dura a animação em segundos. O valor padrão é 1. Valores menores = mais rápido, valores maiores = mais lento e dramático.
0.3s 4s 1.5s
self.play( Create(square), run_time=1.5 )
Impacto visual: run_time baixo (<0.5) cria um aparecimento instantâneo, bom para highlights rápidos. run_time alto (>2) dá peso e drama — ideal para revelar uma métrica importante.

lag_ratio
Quando você anima múltiplos elementos juntos, lag_ratio define o quanto cada elemento atrasa em relação ao anterior. 0 = todos juntos, 1 = um de cada vez, valores intermediários criam cascatas elegantes.
0 1 0.3
self.play( LaggedStart( *[FadeIn(dot) for dot in dots], lag_ratio=0.3 ) )
Impacto visual: lag_ratio = 0 é impactante mas pode parecer estático. lag_ratio = 0.2–0.4 cria o efeito "cascata" mais elegante. lag_ratio = 1 é mais didático, bom para ensinar passos.

〜 Curva de Animação (rate_func)
rate_func
A forma como o movimento acelera e desacelera. É a diferença entre uma animação robótica e uma animação com personalidade. Cada opção muda completamente a sensação.
from manim import * self.play( square.animate.shift(RIGHT*3), rate_func=smooth )
Para PMs: Use smooth para métricas (profissional). Use ease_out para revelar resultados (impactante no início). Evite linear — parece programado demais.

🎨 Visual & Cor
scale(factor)
Multiplica o tamanho do objeto. 1 = tamanho original, 2 = dobro, 0.5 = metade. Pode ser animado com animate.scale() para criar zoom in/out dramáticos.
0.2× 3× 1.5×
self.play( metric_text.animate .scale(1.5) .set_color(GREEN) )
Uso em produto: Scale de 1 → 2 em uma métrica de NPS cria "wow moment". Scale de 1 → 0.5 pode simbolizar redução de churn. Evite valores extremos (>3) pois saem da tela.

set_color_by_gradient()
Aplica um degradê de cor ao longo do texto ou forma. Você passa duas ou mais cores e o Manim interpola automaticamente. Muito usado em tipografia cinética.
title = Text("GROWTH") title.set_color_by_gradient( ORANGE, PINK )
Para apresentações: Azul/ciano = confiança técnica. Laranja/pink = energia e crescimento. Verde/lima = resultado positivo. Evite degradê vermelho em métricas boas — associação negativa.

🌀 Spring Physics
damping
Amortecimento da mola. Controla o quanto o elemento "quica" antes de parar. Baixo = quica muito (elástico). Alto = para direto (comportamento rígido). Valor padrão da maioria dos casos: 12–20.
1 50 12
const y = spring({ frame, fps, from: 60, to: 0, config: { damping: 12, stiffness: 100, mass: 1, } });
Damping 5–8: Muito elástico, jovial — bom para apps e produtos consumer. Damping 12–20: Natural, polido — ideal para cases de produto. Damping 30+: Sem bounce, direto — adequado para apresentações corporativas sérias.
🌀 Controla elasticidade
PREVIEW · damping
stiffness
Rigidez da mola. Controla o quão rápido o elemento chega ao destino. Baixo = mola frouxa (lento, fluido). Alto = mola rígida (rápido, snappy). Combinado com damping, define a personalidade completa.
10 300 100
const y = spring({ frame, fps, config: { damping: 12, stiffness: 100, mass: 1, } });
Stiffness 30–60: Animação suave e "fluida" — bom para transições de tela. Stiffness 100–150: Equilíbrio perfeito — padrão para elementos de UI. Stiffness 200+: Muito snappy — pop-ups e notificações.
⚡ Controla velocidade de chegada
PREVIEW · stiffness
mass
Massa do objeto. Quanto mais pesado, mais inércia. Mass baixo = leve, reage rápido à mola. Mass alto = pesado, começa devagar mas vai longe. Pense num objeto de papel vs. de metal.
0.1 5 1.0
const scale = spring({ frame, fps, from: 0, to: 1, config: { damping: 12, stiffness: 100, mass: 1.0, } });
Mass 0.1–0.5: Elementos leves — ícones, badges, labels. Mass 1: Padrão universal. Mass 2–5: Elementos "pesados" — modais grandes, hero cards. Aumenta dramatismo de elementos importantes.

📐 interpolate()
inputRange (frame range)
Define em quais frames a animação acontece. [0, 30] significa "anime do frame 0 ao 30". Frames antes = valor inicial, frames depois = valor final. Isso controla o timing preciso dentro do vídeo.
INÍCIO (frame)
0 60 0
FIM (frame)
10 120 45
const opacity = interpolate( frame, [0, 45], [0, 1], { extrapolateRight: 'clamp' } );
Range curto (10–20 frames): Animação rápida, snappy. Range longo (60–90 frames): Fade suave e elegante. Use Sequence delay para escalonar o inputRange de cada elemento sem calcular na mão.

Easing (extrapolate + curve)
Como o valor muda dentro do range. O Remotion usa funções de easing do pacote @remotion/easing. Cada curva muda completamente a sensação da transição.
import { Easing } from '@remotion/easing'; const x = interpolate(frame, [0, 60], [0, 400], { easing: Easing.inOut(Easing.ease), extrapolateRight: 'clamp', });
Para PMs: easeInOut para qualquer transição profissional. easeOut para revelar resultados positivos (chega forte, para suave). easeInBack para elementos que "saem" do slide de forma teatral.

〜 Suavização de Movimento
lerp() — fator de suavização
Linear interpolation — a função mais importante do p5.js. Cada frame, o objeto se move X% do caminho até o destino. Fator baixo = movimento lento e suave. Fator alto = chegada rápida. Nunca chega exatamente ao destino com valores <1.
0.01 0.5 0.08
// Chamado 60x por segundo no draw() x = lerp(x, targetX, 0.08); y = lerp(y, targetY, 0.08); // quanto menor o valor, // mais "borracha" o movimento
0.01–0.04: Movimento fluido como líquido — bom para partículas e fundo. 0.06–0.12: Responsivo mas suave — padrão para cursores e elementos interativos. 0.2–0.5: Snappy — boa para hover states e highlight imediato.

Trail Alpha (rastro)
Em vez de limpar o canvas com background() completamente, você pinta um fundo semi-transparente. Isso cria um rastro. Quanto menor o alpha, maior o rastro — parece que o objeto "deixa uma trilha".
5 255 40
function draw() { // alpha baixo = rastro longo background(0, 40); // alpha = 255 = sem rastro ellipse(x, y, 20); }
Alpha 5–20: Rastro longo, estético — bom para arte generativa e fundos. Alpha 40–80: Rastro médio — ótimo para partículas de usuário em demos de UX. Alpha 150+: Rastro curto ou sem rastro — para visualizações de dados onde clareza é prioridade.

⚡ Física de Partículas
Força (attraction / repulsion)
Quanto o mouse puxa ou empurra as partículas. Um vetor de força é calculado com base na distância. Força positiva = atração. Força negativa = repulsão. Controla o quanto sua "feature" influencia o comportamento dos usuários.
← Repele Atrai → +0.8
let dx = mouseX - p.x; let dy = mouseY - p.y; let dist = sqrt(dx*dx + dy*dy); if (dist < radius) { let force = 0.8; p.vx += (dx/dist) * force; p.vy += (dy/dist) * force; }
Atração (+0.5 a +2): Simula usuários "atraídos" por uma feature. Ótimo para demos de onboarding. Repulsão (-0.5 a -2): Simula atrito ou friction point. Use para mostrar onde usuários fogem no user journey.

noiseScale — Perlin Noise
Ruído de Perlin cria movimento orgânico — como folhas no vento, não robótico. O scale controla o "zoom" no campo de ruído. Scale baixo = movimento amplo e suave. Scale alto = movimento rápido e caótico.
0.001 0.05 0.008
let angle = noise( x * 0.008, y * 0.008, t * 0.003 ) * TWO_PI * 4; p.vx += cos(angle) * 0.3; p.vy += sin(angle) * 0.3;
0.002–0.005: Partículas se movem em grandes curvas suaves — bom para fundos e ambientes. 0.01–0.02: Movimento orgânico médio — ideal para user flows e demos. 0.03+: Turbulência caótica — representa fricção ou caos no sistema.

🎨 Cor & Visual
colorMode + HSB
O p5.js tem dois modos de cor: RGB (vermelho, verde, azul) e HSB (matiz, saturação, brilho). HSB é infinitamente mais útil para animação porque permite girar o hue para criar arco-íris, e controlar saturação para fade natural.
HUE BASE (0–360°)
0° 360° 200°
SATURAÇÃO (0–100%)
0% 100% 80%
HUE RANGE (arco de cores)
0° 180° 60°
colorMode(HSB, 360, 100, 100); let hue = 200 + (i/total) * 60; fill(hue, 80, 90); ellipse(x, y, r);
Saturação 0: Tons de cinza — elegante para fundo. Saturação 60–80: Cores vibrantes mas não agressivas — padrão para produto. Hue range 0: Cor única sólida. Hue range 120+: Paleta ampla e diversa.
