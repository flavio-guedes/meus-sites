01 — O Problema Real

Por que agências de
marketing travam.

O gargalo humano

Em agências criativas, o fundador ou diretor acaba sendo o único ponto de conexão entre comercial, estratégia, criação e entrega. Quando ele está dentro da operação, a agência cresce até o limite da sua disponibilidade — e então trava.

"O problema não está na qualidade do trabalho — está em quem precisa estar presente em cada etapa para o trabalho acontecer."

Diagnóstico que originou o projeto
O que ferramentas existentes não resolvem

ChatGPT, Gemini e similares são excelentes para tarefas isoladas. Mas nenhum deles tem memória nativa do que o agente anterior produziu. Cada conversa começa do zero. Para simular um time, é preciso arquitetar o fluxo de informação manualmente.

Por que construir em vez de usar plataforma pronta

Ferramentas como Make e n8n resolvem o fluxo, mas abstraem o controle sobre os prompts. Construir na API diretamente deu controle total sobre o que cada agente recebe, como processa e o que entrega — a habilidade central de um AI Specialist.

02 — A Solução

Sequential pipeline com
human-in-the-loop.

O padrão adotado

O padrão adotado é o Sequential Pipeline: cada agente recebe como contexto o output completo do agente anterior, criando uma cadeia de raciocínio acumulativa. Em dois pontos estratégicos, o pipeline pausa para aprovação humana antes de continuar.

Por que pausar para aprovação humana?

Os agentes 02 e 04 pausam o pipeline porque são os momentos de maior impacto financeiro e criativo. Uma proposta aprovada define o escopo inteiro. Uma criação aprovada orienta toda a execução. IA executa — humano decide nas bifurcações que importam.

Como o contexto é preservado entre agentes

Cada output é salvo em memória e injetado no prompt do próximo agente. O agente 05 de Mídia Paga recebe o briefing criativo do 04 — e não precisa inferir nada. É o que diferencia uma cadeia de agentes de 7 perguntas soltas num chat.

03 — Prompt Design

Como cada agente
foi instruído.

Anatomia do prompt

A eficácia de um sistema multi-agente depende quase inteiramente da qualidade dos prompts. Cada agente foi projetado com quatro componentes: papel, contexto, tarefa e formato de saída esperado.

Agente 03 — Estratégia · Prompt completo
// Papel do agente Você é um estrategista de marketing digital especializado em agências criativas. Seu trabalho é traduzir uma proposta comercial aprovada em um plano estratégico executável. // Contexto acumulado Com a seguinte proposta aprovada pelo cliente: [output do Agente 02]. Para o cliente descrito originalmente como: [input do usuário]. // Tarefa específica Crie um plano estratégico com: canais prioritários (orgânico + pago), pilares de conteúdo (3-4), frequência de publicação, tom de comunicação e métricas de sucesso. // Formato de saída Responda em português brasileiro. Seja direto e prático — sem introduções genéricas. Estruture em tópicos claros que o próximo agente (Criação) possa usar como briefing.
A instrução que faz o sistema funcionar

A linha mais importante de cada prompt não é a tarefa — é a instrução de formato direcionada ao próximo agente. O Agente 03 é instruído a estruturar o output "de forma que o próximo agente possa usar como briefing". Isso cria coerência sem memória compartilhada.

O que testei e descartei

Prompts genéricos como "Crie uma estratégia de marketing" geravam outputs úteis mas desconectados do cliente real. Adicionar o output anterior como contexto explícito aumentou drasticamente a especificidade — e foi a principal iteração do projeto.

05 — Limitações & Próximos Passos

O que eu melhoraria
na próxima versão.

Honestidade técnica

Reconhecer as limitações de um sistema é tão importante quanto construí-lo. Aqui estão as três evoluções já identificadas.

Limitação atual
Sem memória entre sessões

Cada vez que o sistema é iniciado, começa do zero. O histórico do cliente não é preservado. A próxima versão usaria RAG (recuperação de documentos) para contextualizar sessões anteriores com histórico real do cliente.

Limitação atual
Outputs sem validação automática

O sistema não verifica se o output do agente anterior está dentro do esperado antes de passar adiante. Um agente validador poderia checar qualidade e solicitar revisão quando necessário, reduzindo erros em cascata.

Limitação atual
Agentes sem especialização fine-tuned

Todos os agentes usam o mesmo modelo base. Em uma versão avançada, o agente de Mídia Paga usaria um modelo fine-tuned com dados reais de campanhas para recomendações mais precisas de segmentação e budget.
