Conceitos Fundamentais
Analogia — Go é como uma fábrica moderna
Imagine uma fábrica onde cada trabalhador (goroutine) executa sua parte ao mesmo tempo, comunicando-se por esteiras (channels), sem conflitos. O gerente (runtime do Go) decide quem trabalha em qual máquina e quando.
Package (Pacote)
Todo código Go fica dentro de um pacote. É como uma pasta temática — você agrupa funções relacionadas. O pacote especial main é onde o programa começa a executar.
gopackage main
// ponto de entrada do programa
Função (Function)
Uma função é um bloco de ações que você pode chamar pelo nome. Recebe entradas, executa algo, devolve um resultado. A função main() roda automaticamente ao iniciar.
func soma(a, b int) int {
  return a + b
}
Variável e Tipo
Uma variável é uma caixinha com nome que guarda um valor. Em Go, toda variável tem um tipo — número, texto, booleano. O operador := cria a variável e já define o tipo sozinho.
var idade int = 25
nome := "Ana" // tipo inferido
Loop (Laço)
Em Go existe apenas o for para repetições — substitui o while e o do-while. É como um robô que repete uma tarefa enquanto a condição for verdadeira. Simples e poderoso.
for i := 0; i < 5; i++ {
  fmt.Println(i)
}
Struct (Estrutura)
Uma struct é como um formulário personalizado — você define quais campos ela tem. Quer guardar dados de um usuário? Crie uma struct Usuario com Nome, Email e Idade.
type Usuario struct {
  Nome string
  Idade int
}
Error Handling (Erros)
Go não usa exceções como Java. Funções retornam dois valores: o resultado e um possível erro. Você é obrigado a verificar o erro — Go não deixa você ignorar problemas.
val, err := abrirArquivo("x.txt")
if err != nil {
  fmt.Println("erro!")
}

Goroutines e Concorrência
Analogia — Cozinha de restaurante
1 cozinheiro (thread) faz tudo em sequência. Goroutines são como ter 10 cozinheiros simultâneos — cada um faz sua parte ao mesmo tempo, comunicando-se pelo balcão de pedidos (channel). O resultado chega muito mais rápido.
Goroutines ativas — simulação
Cada quadrado representa uma goroutine rodando em paralelo
goroutines ativas: 3
Channel — comunicação entre goroutines
O channel é uma esteira compartilhada — uma goroutine envia, a outra recebe
goroutine A
envia
goroutine B
recebe
ch := make(chan string)
go func() { ch <- "mensagem" }()
msg := <-ch // recebe
Mutex — sem conflito de acesso
Imagine duas goroutines escrevendo no mesmo arquivo ao mesmo tempo — daria bagunça. O Mutex é como uma chave de banheiro: só uma goroutine usa o recurso por vez.
var mu sync.Mutex
mu.Lock() // pega a chave
mu.Unlock() // devolve a chave
WaitGroup — sincronização de tarefas
O WaitGroup é como um placar de presença: você registra quantas goroutines vão trabalhar e espera todas terminarem antes de continuar. Igual aguardar todos os cozinheiros para servir a mesa.
var wg sync.WaitGroup
wg.Add(3)
go tarefa(&wg) // wg.Done() no fim
wg.Wait()
Go vs Thread tradicional — por que goroutines são melhores?
Uma goroutine usa ~2 KB de memória. Uma thread do SO usa ~1 MB. Você pode ter milhões de goroutines simultâneas.

História do Go
2007
A frustração do Google
Robert Griesemer, Rob Pike e Ken Thompson — criadores do UNIX — estavam cansados de C++ demorar horas para compilar. Decidiram criar algo melhor.
2009
Go torna-se open-source
Go é anunciado em 10 de novembro de 2009. A mascote Gopher nasce, desenhada por Renée French.
2012
Versão 1.0 — estabilidade garantida
Go 1.0 é lançado com promessa de compatibilidade permanente: código escrito hoje funcionará em versões futuras.
2013
Docker nasce em Go
O Docker, que revolucionou o DevOps, foi escrito em Go. Provou que Go era perfeito para ferramentas de infraestrutura.
2014
Kubernetes — orquestração em Go
O Google lança o Kubernetes em Go. Hoje é o padrão mundial para gerenciar apps na nuvem.
2022
Go 1.18 — Generics chegam
O recurso mais pedido da história: Generics. Permite escrever funções que funcionam com qualquer tipo sem repetição de código.

Por que o nome "Go"?
Simples por filosofia
O nome curto reflete a essência da linguagem: fazer tudo com simplicidade. Go compila em segundos, tem poucos comandos, e o código parece inglês simples. Um dev pode ler código Go de outra pessoa sem precisar de manual.
Go vs outras linguagens
Analogia — Escolhendo o veículo certo
Python é um carro automático — fácil de dirigir, ótimo para cidade. C++ é uma Fórmula 1 — ultra rápido, mas difícil de pilotar. Go é um carro esportivo com câmbio automático — rápido e ainda fácil de usar.

Use Go quando...
Precisa de alta performance com muitas tarefas simultâneas, criando APIs e microserviços, ferramentas de linha de comando ou projetos de infraestrutura cloud.
Evite Go quando...
Precisa de Machine Learning (use Python), quer protótipos ultra-rápidos, necessita de GUI desktop, ou a equipe só conhece Java/OOP clássico.
